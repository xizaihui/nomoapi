### Bug Fix
- **Preview 400 error**: `json.NewDecoder(c.Request.Body).Decode()` fails because Gin's gzip middleware replaces `c.Request.Body`. Fixed by using `c.ShouldBindJSON()` instead — consistent with rest of codebase (e.g., `ratio_sync.go`).

### Lesson Learned
- **Always use `c.ShouldBindJSON()` in Gin controllers** instead of `json.NewDecoder(c.Request.Body)` — the gzip middleware wraps the body and direct reading can fail with 400.
- **`common.OptionMap` is `map[string]string`** (not `map[string]interface{}`), so no type assertion needed — just use values directly as strings.
