### Key Architecture Decisions
- Audit config stored in User.Setting JSON: `audit_enabled`, `audit_webhook`, `audit_rule_group`
- AuditLog in LOG_DB (shared with main DB since no LOG_SQL_DSN), AuditRule in main DB
- All audit ops async via `gopool.Go()` тАФ zero impact on relay latency
- Rule cache with 60s TTL to avoid DB queries per request
- `isRelayPath()` gates audit to only relay endpoints (/v1/chat/completions, etc.)
