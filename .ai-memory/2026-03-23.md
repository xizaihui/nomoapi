## OpenToken 生产环境 502 修复 (23:15 UTC)

### 问题
- `https://api.opentoken.io` 返回 502 Bad Gateway
- 服务器约 1 小时前重启过（uptime 57min）

### 诊断
- Docker 容器正常运行，健康检查 healthy
- 容器内部 `127.0.0.1:3000` API 正常响应
- 宿主机 `localhost:3000` TCP 连上但 0 bytes 返回（10s 超时）
- docker-proxy 进程存在，iptables NAT 规则存在，但数据包不通
- Nginx 错误日志: `recv() failed (104: Connection reset by peer)`

### 原因
Docker 网络栈在服务器重启后未正确恢复。docker-proxy 端口转发表面存在但实际不通。

### 修复
`systemctl restart docker` — 重建网络栈和 docker-proxy，恢复正常。

### 教训
服务器意外重启后，如果遇到 502 且容器显示 healthy，优先考虑 `systemctl restart docker` 而非只 restart 容器。
