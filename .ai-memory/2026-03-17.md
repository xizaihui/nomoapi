The Docker in-container bun build produced different hashes than local build, suggesting environment-dependent behavior. ### What was reverted

### Commits

- `d28203a1` — Revert: restore Semi UI runtime (working)

### Lesson learned
**Do NOT remove Semi UI runtime without thorough browser testing.** The vite-plugin-semi likely injects runtime helpers that other Semi code depends on. The compat layer intercepts `import '@douyinfe/semi-ui'` but doesn't cover everything vite-plugin-semi does internally. Future Semi removal requires: (1) removing ALL Semi packages from node_modules, (2) replacing the compat Chat component with a non-Semi implementation, (3) testing in actual browser, not just build success.

### Docker architecture change (kept)
- Dockerfile no longer runs `bun install` + `bun run build` inside container
- Instead: `web/dist` is pre-built locally, copied via `COPY . .`, embedded by Go's `//go:embed web/dist`
- `.dockerignore` no longer excludes `/web/dist`
- This ensures Docker image matches locally tested build exactly
- Image size: ~108MB on alpine:3.20

## P4-1: @lobehub/icons Isolation — FAILED FIRST, SUCCEEDED SECOND (~10:00-14:25 UTC)

### First attempt — broke admin pages
- Moved 32 named icon imports + getModelCategories/getChannelIcon/renderModelTag from render.jsx to lobe-icons.jsx
- Updated 16 callers to import from helpers/lobe-icons directly
- Build succeeded, headless test passed from server
- **User reported: "页面组件出错了 Failed to fetch dynamically imported module: index-CfqHuL6P.js"**

### Root causes found (TWO Go-level bugs):
1. **`go:embed web/dist` excludes underscore-prefixed files** — Vite generates chunks like `_arrayReduce-CMvE2-Ut.js` which returned 404. Fix: `//go:embed all:web/dist`
2. **`gin-contrib/gzip` sets wrong Content-Length (23 bytes)** on gzipped responses from embed.FS, corrupting browser downloads. Fix: disable gzip on web static file router (API routes keep gzip)

### Rollback and re-do
- Rolled back P4-1 frontend changes but kept Go fixes
- User confirmed rollback worked
- Re-applied P4-1 with Go fixes in place — all tests passed
- User confirmed: "这次没问题"

### Final commit: `107ad9f1`
- Main entry: 1,744KB → 1,267KB (-27%)
- lobe-icons: 3,604KB isolated chunk (lazy-loaded on admin pages only)
- Go: `all:web/dist` + no gzip on web router

### Critical lesson learned
**Go's `go:embed` directive excludes files starting with `_` or `.` by default.** Must use `all:` prefix. This was a pre-existing bug that only manifested when Vite code-splitting produced underscore-prefixed chunks.
